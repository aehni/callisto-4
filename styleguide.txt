Naming:
- All Files are named in UpperCamelCase
- Avoid duplicate namings if not necessary:
	e.g. 'view/Item/SingleItemView.twig' => 'view/Item/SingleItem.twig'
	or 	'js/src/app/components/Login/LoginVueComponent.js' => 'js/src/app/components/Login/Login.js'
- Names of files or directories should describe usage/ function of file in correct way:
	e.g. 'PaymentShippingService.twig' => 'PaymentShippingList.twig' (Twig cannot provide services)
- Vue.js-Components are stored as 'resources/js/src/app/components/<COMPONENT_NAME>/<COMPONENT_NAME>.js'
  Depending templates are stored as 'resources/views/Templates/<COMPONENT_NAME>/<COMPONENT_NAME>.twig'
  Do not use <template>-Tag. Use <script type="x/template"> instead.
  Template-IDs has to be valid HTML-selectors and must be unique. Use format: 'vue-<COMPONENT_NAME_SNAKE_CASE>'
- Twig-Templates are stored in 'resources/views' and should be grouped by usecases:
	-> Blog/		for templates related to blog contents
	-> Checkout/	templates required for checkout (including basket)
	-> Content/		templates for displaying static content
	-> Item/		templates or snippets to display item data (containing also list items included in Item-Category-Lists)
	-> PageDesign/	global templates for basic page design, e.g. header, footer, newsletter etc.
  If a single layout-component requires multiple twig-files (e.g. parent-/child-templates, macros), the should be grouped in subfolders
